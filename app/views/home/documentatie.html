1. Introduction

1.1 Purpose

Terrorism Web Visualiser(TeW) is a tool designed to allow flexible visualisation of global terrorism related data.

2. Overall Description

2.1 Product Perspective

TeW is a new, self-contained product, that is not based on any previous projects.

2.2 Product Functions

The web application is designed to contain a large, flexible database of terrorist attacks, that can be easily updated through the administration UI. The statistics function of the application uses said data, along with various filters for the attacks, and dynamically generates statistics and graphs regarding the given data. The map function is used to carthographically locate events for better visualisation, by also using the database along with various user given filters.

2.3 User Classes and Characteristics

This web application provides a visualiser for terrorism around the world; linked to a database containing over 18k registered attacks, this application can be used in order to easily explore this database. The application can be accesed by two types of users: administrator and client. The client can be anyone who has a particular interest in terrorism at global scale, someone who wants to easilly explore the database and possesess a medium-level of understanding into what terrorism is. The administrator is a priviledged user; this kind of user can access an administration page in order to change the content shown; he can edit or delete the articles and also he can add new articles. This kind of user needs to have high-level knowledge regarding the consequences of terrorism, a great sense and capability of sorting the attacks by certain criteria.

3. External Interface Requirements

3.1 User Interfaces

The non-priviledged user will be able to see and interact with 4 pages (Map, Home, Statistics, Contact), through which can easily navigate using the navigation bar. If they press the button of the same page they are in, it will act like a back-to-the-top button. Administrators will be able to access 2 other pages (Admin, Administration), one used for logging in and the other to manage the aplication. Each page has its own purpose:

3.1.1 Admin

Admin page contains a form where the user is asked to input a username and a password. These credentials will be sent to backend, to be verified. The purpose of the page is to redirect admins to the administration page, after they are logged in.

3.1.2 Administration

Administration page can be accessed after the admin introduces correct credentials in the Admin form. It contains a button with the option to add a new article, a searchbar, along with various filters for looking up articles on the left side of the page, and the articles resulted after the search on the right side of the page with a short description. The articles can be edited, deleted or visualised using 1 of the 3 buttons below them. The purpose of the page is to be used by the administrator to add, delete or edit articles.

3.1.3 Contact

Contact page contains a navigation bar with 4 buttons that redirect the user to any of the 4 pages avaible to them (Map, Home, Statistics, Contact), contact information, a short description and a photo of each participant to the project. The purpose of the page is to gather information about the project creators.

3.1.4 Home

Home page is the main page of the project, the page the users will first see. The page contains a navigation bar with 4 buttons that redirect the user to any of the 4 pages avaible to them (Map, Home, Statistics, Contact), a signifiant banner, important information that will be added from the backend and will replace the actual ones (graphics, stats, map). It contains information about terrorism to inform the users about the theme of the project. The purpose of the page is to greet the user.

3.1.5 Map

Map page contains a map where the terorist atacks will be highlighted depending on the user-controlled filters. The page contains a navigation bar with 4 buttons that redirect the user to any of the 4 pages avaible to them (Map, Home, Statistics, Contact), a search bar, along with filters, where information will be introduced and then sent to backend, to locate and pinpoint terorist atacks on the map. The purpose of the page is to find and display the atacks the user is interested in.

3.1.6 Statistics

Statistics page contains a navigation bar with 4 buttons that redirect the user to any of the 4 pages avaible to them (Map, Home, Statistics, Contact), a searchbar, filters where the user will introduce information that will be sent to backend, necesarry to create the statistics, a short description of the articles about the atacks involved in the statistic with a button that allows the user to see the entire article. Below the articles the graphic and statistics will be generated. The purpose of the page is to create the graphics and statistics the user is interested in, with some additional relevant data.

4. System Features

4.1 Administration of database via the administration page

4.1.1 Description and Priority

High priority: This feature is responsible for making active changes to the database while the webserver is active. The database is also responsible for generating statistics, charts and maps.

4.1.2 Stimulus/Response Sequences

The administrator logs in via the admin page and is presented with the admin panel. There, he can manage the database via the "Add new" button to make insertions, or via the "Edit" and "Delete" buttons corresponding to each article, to update or delete.

4.1.3 Functional Requirements

The user is required to input valid credentials in order to access the admin panel. The database must be up and running in order for the queries to take effect.

4.2 Searching and filtering results from the database

4.2.1 Description and Priority

Medium priority: This feature is responsible for querying the database, in order to select articles relevant to the user-inputted filters.

4.2.2 Stimulus/Response Sequences

The user inputs keywords relevant to the articles they wish to discover, and performs further filtering via the filter form. The selected results will be shown on the page.

4.2.3 Functional Requirements

The database must be up and running in order for the queries to take effect.

4.3 Creating and downloading maps and charts

4.3.1 Description and Priority

High priority: This feature is responsible for generating various maps and charts based on the search filters, along with exporting the generated charts and maps for further processing.

4.3.2 Stimulus/Response Sequences

The generation process is done via the same search field as the one in feature 4.2. The user is able to export the generated figures in various formats via a download button and use them for later processing.

4.3.3 Functional Requirements

The database must be up and running in order for the queries to take effect.
